🧠 Software Engineering Principles & Best PracticesCe document regroupe les principes fondamentaux de conception logicielle appliqués dans mes projets (Java 21, Spring Boot, Architecture Hexagonale).🏗️ SOLID PrinciplesLes 5 piliers de la conception orientée objet pour un code maintenable.PrincipeDescriptionSRPSingle Responsibility Principle : Une classe ne doit avoir qu'une seule raison de changer.OCPOpen/Closed Principle : Ouvert à l'extension, fermé à la modification.LSPLiskov Substitution Principle : Les sous-classes doivent être substituables par leur classe de base.ISPInterface Segregation Principle : Préférer plusieurs interfaces spécifiques à une seule interface généraliste.DIPDependency Inversion Principle : Dépendre des abstractions, pas des implémentations (fondement de l'architecture hexagonale ).💡 Pragmatic Design (DRY, KISS, YAGNI)Lutter contre la complexité et la duplication.DRY (Don't Repeat Yourself) : Chaque connaissance doit avoir une représentation unique dans le système. Évite la duplication logique.KISS (Keep It Simple, Stupid) : La simplicité est la sophistication suprême. Un code simple est plus facile à tester et à maintenir.YAGNI (You Ain't Gonna Need It) : Ne développez pas une fonctionnalité tant qu'elle n'est pas nécessaire. Évitez l'over-engineering.⚡ Data & Transactions (ACID)Garantir la fiabilité des données, crucial pour les systèmes de paiement ou de fidélité.Atomicity : Tout ou rien. Si une étape échoue, toute la transaction est annulée.Consistency : La transaction amène le système d'un état valide à un autre état valide.Isolation : Les transactions s'exécutent comme si elles étaient seules (gestion de la concurrence).Durability : Une fois validée, la donnée est persistée de manière permanente, même en cas de crash.🧩 GRASP PatternsGeneral Responsibility Assignment Software Patterns.Information Expert : Assigner la responsabilité à la classe qui détient l'information.Creator : Déterminer qui doit créer une nouvelle instance d'un objet.Low Coupling : Réduire l'interdépendance entre les modules pour faciliter les évolutions.High Cohesion : Garder les responsabilités d'un module fortement liées et focalisées.🛠️ Application Concrète (Tech Stack)Java 21 & Spring Boot 3 : Utilisation des Records pour l'immuabilité (SRP) et de l'injection de dépendances (DIP).Architecture Hexagonale : Isolation du Domain (Métier) des interfaces (API, DB, Kafka) pour garantir le couplage faible.TDD/BDD : Validation systématique du comportement par les tests.Note : Ce document sert de base pour les revues de code et le mentoring technique.
